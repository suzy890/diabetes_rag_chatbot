# 고령 당뇨 환자를 위한 넛지 · RAG 웹앱

> 고령 제2형 당뇨 환자의 건강행동을 돕기 위해, **선제적 넛지**와 **근거 기반 RAG 채팅**을 하나의 대화 화면에서 제공하는 연구용 반응형 웹앱.

---

## 프로젝트 목적

이 프로젝트는 새로운 AI 모델을 개발하거나 AI의 기술적 성능을 경쟁시키는 것이 목적이 아니다.
근거 기반 정보 제공(RAG)과 행동 넛지가 실제 사용자의 **이해 · 신뢰 · 자기효능감 · 행동의도 · 행동 수행**에
어떤 관계를 갖는지 살펴보는 **사회 · 행동과학적 실증 연구**다.

최종 산출물은 웹앱뿐 아니라 **분석 가능한 연구 데이터**다. 따라서 화면보다 데이터 구조를 먼저 설계한다.

## 대상 사용자

- **주 사용자:** 고령 제2형 당뇨 환자 (연구 참여자)
- **부 사용자:** 연구자 (참여자 등록, 데이터 확인 및 분석)
- 실험군 · 대조군을 나누지 않고, 모든 참여자가 동일한 연구용 웹앱을 사용하는 **단일군 현장 실증** 설계

## 주요 기능

| 기능 | 설명 |
|------|------|
| **선제적 넛지** | AI가 먼저 말을 걸어 작은 건강행동(식사 · 운동 · 복약 · 혈당관리)을 떠올리고 선택 · 실행하도록 돕는다. 승인된 규칙과 템플릿을 우선 사용한다. |
| **RAG 당뇨 채팅** | 사용자의 질문을 승인된 신뢰 문서에서 검색한 뒤, 그 근거 범위 안에서 답하고 출처를 함께 제시한다. |
| **하나의 채팅 화면** | 넛지와 자유 질문이 하나의 대화 흐름 안에서 결합된다. |
| **연구 데이터 기록** | 참여자 · 세션 · 질문 · 답변 · 검색 근거 · 넛지 · 행동 반응 · 설문 · 시점 · 버전 · API 비용 · 오류를 모두 저장한다. |

## 사용 예정 기술

- **언어:** Python
- **웹 화면:** Streamlit
- **데이터베이스:** Supabase (PostgreSQL)
- **벡터 검색:** Supabase pgvector
- **AI 응답:** 외부 LLM API
- **임베딩:** 외부 Embedding API
- **버전 관리:** GitHub

> 초기 버전에서는 LangChain 등 대형 프레임워크를 도입하지 않고, 웹앱 화면 ↔ Supabase ↔ 외부 API를 직접 연결하는
> **모듈형 단일 애플리케이션**으로 구현한다. 핵심 실행 Python 코드는 500줄 이내를 목표로 한다.
