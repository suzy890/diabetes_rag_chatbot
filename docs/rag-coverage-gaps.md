# RAG 커버리지 갭 리포트 (1차)

> **작성일:** 2026-07-16 · **근거:** `retrieval_logs` + `retrieval_chunks` 실제 검색 기록
> 목적: "부분·부족" 판정이 난 질문을 모아 **어떤 주제의 근거 문서가 부족한지**를 데이터로 찾는다.
> 이 리포트는 감이 아니라 **로그 기반**으로 다음에 추가할 문서를 정하기 위한 것이다. (RAG_RULES §3 "보류율 모니터링")

## 판정 기준 (현재 잠정값 — U4)

| 판정 | 조건(최상위 청크 코사인 유사도) | 답변 |
|------|------|------|
| 충분 | ≥ 0.45 | 근거로 정상 답변 |
| 부분 | 0.30 ~ 0.45 | 일반 정보 + "의료진 상의" (단편적으로 느껴질 수 있음) |
| 부족 | < 0.30 | 보류 |

## 1. 잘 답하는 주제 (충분 — 유사도 0.45↑)

현재 4개 문서로 **충분히 커버되는** 주제다. 문서 추가 불필요.

- 당화혈색소 검사 주기 (0.52)
- 발 관리 / 맨발 금지 (0.55)
- 저혈당이 왔을 때 대처 (0.60)
- 약·인슐린을 빠뜨렸을 때 (0.46)

## 2. 약하게 답하는 주제 (부분·부족) → 문서 추가 대상

**참여자가 실제로 물어본** 질문 중 근거가 약했던 것들이다. (배포 데모에서 수집)

| 참여자 질문 | 판정 | 유사도 | 부족한 근거 |
|------|------|------|------|
| "점심 메뉴로 칼국수를 먹으려는데 괜찮니?" | 부분 | 0.42 | **구체적 식품·외식 선택** — 지금 문서엔 "골고루·탄수화물 조절" 같은 원칙만 있고 특정 음식 안내가 없음 |
| "오늘 아침 공복혈당이 142인데 괜찮은건가?" | 부분 | 0.36 | **혈당 수치 해석·목표범위** — 진단기준(126)은 있으나 "이 수치면 무슨 의미/무엇을 할지"가 약함 |
| "갑자기 어지럽고 식은땀 나면 뭘 먹어야?" | 부분 | 0.36 | **저혈당 증상 알아채기** — 대처법은 있으나 증상 인식 근거가 약함 |

## 3. 추천 — 다음에 추가하면 좋을 문서 (타깃)

양보다 **적중**이 중요하다. 위 갭에 맞는 문서를 우선한다.

1. **당뇨병 식사·식품 가이드** (식품교환표, 외식·메뉴 선택, 면류·간식)
   → "칼국수 먹어도 되나?" 같은 **일상 식품 질문**을 충분으로 끌어올림
2. **혈당 자가관리·목표 가이드** (공복/식후 혈당 목표 범위, 수치별 일반 안내)
   → "공복혈당 142" 같은 **수치 해석 질문**을 보강
3. **(선택) 저혈당 증상·응급 카드** (증상 목록·초기 신호)
   → "어지럽고 식은땀" 같은 **증상 인식 질문** 보강

## 4. 반드시 기억할 것 (문서를 넣어도 안 바뀌는 부분)

- **"내 혈당 142가 괜찮은가?"류 개인 판단**은 문서를 늘려 유사도가 올라가도, 시스템이 "괜찮다/아니다"라고 **단정하면 안 된다**(의료안전 규칙). 일반 정보 + 의료진 권고로 가는 것이 **정상**이다.
- 즉 단편적 답변의 일부는 **버그가 아니라 안전 설계**다. 문서 추가로 개선되는 건 "**일반 정보의 풍부함**"이지 "개인 진단"이 아니다.

## 5. 운영 방법 (파일럿에서)

- 파일럿 중 **'부분·부족' 질문을 주기적으로 이 방식으로 집계**한다.
- 자주 나오는 주제 순으로 **타깃 문서를 추가**하고, 추가 후 같은 질문의 판정이 오르는지 확인한다.
- 판정 임계값(0.45/0.30)도 **함께 튜닝**한다(U4). 문서가 늘면 임계값 조정이 필요할 수 있다.
