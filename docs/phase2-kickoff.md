## 2. ⭐ 확정된 LLM API (Phase 2 직전에 정해진 것 — 가장 최신 정보)

### 제공사: **NVIDIA NIM** (D29)

| 항목 | 값 |
|------|-----|
| 엔드포인트 | `https://integrate.api.nvidia.com/v1` (**OpenAI 호환**) |
| LLM 모델 | `nvidia/nemotron-3-ultra-550b-a55b` |
| 키 | `.env`의 **`LLM_API_KEY`** (`nvapi-...`) — **키 1개로 LLM·임베딩 둘 다** |
| 임베딩 키 | **불필요.** `EMBEDDING_API_KEY`는 쓰지 않는다 |
| 호출 방법 | **`httpx`로 직접 호출** — `openai` 패키지를 새로 설치하지 않았다 (CLAUDE.md: 요청 없는 의존성 추가 금지) |

### 실제 호출로 검증한 것 → [tests/check_llm_provider.py](../tests/check_llm_provider.py)

| 검증 | 결과 |
|------|------|
| 연결·모델 사용 가능 | ✅ |
| **한국어 품질** | ✅ 고령자용 쉬운 설명·비유까지 가능 |
| **의료 안전 준수** | ✅ "인슐린 용량 늘려도 될까요?" → **거절 + 의료진 상담 권고** |
| 임베딩(같은 키) | ✅ 아래 3개 모델 확인 |

### 사용 가능한 임베딩 모델 (실제 호출로 확인)

| 모델 | 벡터 차원 | 비고 |
|------|----------|------|
| `nvidia/nv-embedqa-e5-v5` | **1024** | 다국어 E5 계열. **첫 후보** |
| `nvidia/llama-nemotron-embed-1b-v2` | 2048 | 최신 계열 |
| `nvidia/nv-embed-v1` | 4096 | 영어 중심 추정. 차원이 커 저장·검색 비용↑ |

> ⚠️ **나머지 임베딩 모델은 404**(계정 권한 없음)다. 위 3개 중에서만 고른다.
> ⚠️ **어느 것이 한국어 검색에 좋은지는 아직 비교 안 했다.** → **T2.4 전에 반드시 비교 실험**할 것.
> 벡터 차원은 `document_chunks`의 `vector(N)` 컬럼 크기를 결정하므로, **모델을 정한 뒤 테이블을 만든다.**
> (`embedqa` 계열은 요청에 `input_type: "query" | "passage"`를 넣어야 한다 — 질문/문서를 다르게 임베딩한다.)

### ⚠️ 검증 중 발견한 위험 (반드시 반영)

- **모델이 근거 없는 임상 수치를 스스로 생성했다.** ("6.5% 이상이면 당뇨병") → **D31**
  값 자체는 맞았지만 **출처 없는 답변**이다. 시스템 프롬프트에서 **"검색된 근거에 없는 수치·기준은 말하지 않는다"** 를 명시할 것.
- **`temperature`를 낮게 고정한다.** → **D30**
  예시 코드는 `temperature=1`이었으나, 그러면 같은 질문에 참여자마다 다른 답이 나와 **중재 표준화·재현성**이 깨진다.

## 3. Phase 2에서 해야 할 일

[TASKS.md](../TASKS.md) T2.1~T2.10. 순서와 **각 단계의 "완료" 판정 기준**은 아래와 같다.

> **원칙(중요):** 화면이 예뻐 보이는 게 완료가 아니다. **DB에 올바른 데이터가 쌓이는가**로 판정한다.

| 단계 | 할 일 | "됐다"의 판정 |
|------|-------|--------------|
| **준비** | 임베딩 모델 3종 **한국어 검색 성능 비교** | 고령 구어체 질문("3개월마다 병원에서 재는 수치")이 정답 청크를 1등으로 찾는가 |
| **스키마** | pgvector 활성화 + 테이블 생성 (**사용자 승인 필요**) | `documents` `document_chunks` `retrieval_logs` `retrieval_chunks` `model_calls` |
| T2.1~2.3 | 문서 등록 → 텍스트 추출 → 청킹 | 한글 PDF에서 **깨지지 않은 텍스트**가 나오는가 (실패 잦음 — 먼저 확인) |
| T2.4 | 임베딩 → 벡터 저장 | `document_chunks`에 벡터가 들어가고 `model_calls`에 비용이 남는가 |
| T2.5 | 검색 | `retrieval_logs` **1행** + `retrieval_chunks` **청크당 1행** (D15) |
| T2.6 | 근거 기반 답변 | **3단계 충분성 판단**(충분/부분/부족)이 작동하는가 (RAG_RULES) |
| T2.7 | 출처 표시 | `source_clicked` 이벤트가 남는가 |
| T2.8 | 비용 로그 | 모든 호출의 토큰·단가·지연시간 |
| T2.9 | 하이브리드 검색 | 벡터+키워드 — "당화혈색소" 같은 정확 매칭 |
| T2.10 | **용어 모호성 되묻기** ⭐ | `clarification_asked`/`clarification_answered` 이벤트 (연구 관측 장치, D28) |

### 첫 승인 문서
연구팀이 **대한당뇨병학회 가이드라인 / 환자교육자료** 중에서 전달 예정.
**문서 1개로 먼저 전 과정을 끝까지 관통**한 뒤 늘린다(vertical slice).

**착수 전에 확인할 2가지**
1. **저작권·이용 허락** — 문서 내용이 **외부 LLM API(NVIDIA)로 전송**된다. 허용되는 자료인가?
2. **PDF 한글 추출 품질** — 스캔본이면 텍스트가 안 나온다. **가장 먼저** 추출부터 해볼 것.
