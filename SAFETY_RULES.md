# SAFETY_RULES.md — 건강정보 안전 규칙

> **문서 버전:** v0.1 · **작성 기준일:** 2026-07-09
> 이 문서는 위험/제한 질문을 **어떻게 판별하고 응답하는지**를 정의한다. 상위 원칙은 [.claude/rules/medical-safety.md](.claude/rules/medical-safety.md).
> 고령 당뇨 환자를 다루므로 **안전이 정확성보다 우선**한다. 다만 **안전 응답이 과하게 남발되지 않도록** 범주를 좁고 명확하게 정의한다.
> **모든 고정 안내문 문구는 의료전문가 검토를 받은 뒤 확정한다(현재 초안).**

---

## 1. 제한 대상 (고위험·제한 질문)

다음은 일반 RAG 답변 대신 제한·안전 처리한다.

- 약물 **용량 변경** 질문 (늘리기/줄이기)
- **복약 중단·추가 복용** 질문
- 개인별 **진단** 요청 (예: "이 증상 병명이 뭐예요?")
- **합병증 진단** 요청
- **응급 가능성**이 있는 증상 (예: 의식저하·심한 저혈당·흉통 등)
- 근거 없는 **완치 주장** 요청
- **위해 가능성**이 있는 행동 요청

## 2. 질문 분류와 처리 (5범주)

> **연구팀 피드백 반영(과다 트리거 방지):** 일반적인 자기관리 질문이 안전필터에 **잘못 걸리지 않도록**(false positive 최소화) 고위험 범주를 좁게 잡는다. 안전 응답은 **진짜 위험**에만 뜨게 한다.

| 범주 | 예 | 처리 |
|------|-----|------|
| **① 일반 정보** | "당뇨에 바나나 먹어도 되나요?" | 승인 문서 검색 → 근거 기반 답변 + 출처 (정상 흐름) |
| **② 근거 부족** | 지식베이스에 없는 세부 질문 | 추측 금지, 근거 부족 안내 (RAG 3단계 — 부분 근거면 일반 정보+한계 명시) |
| **③ 의료진 확인 필요** | "제 약을 아침에 먹을까요 저녁에 먹을까요?" | 일반 정보 + **의료진·약사 상담 권고** (완전 거절 아님) |
| **④ 고위험** | "혈당이 너무 낮은데 어떻게 해야 하나요(응급)", "인슐린 용량 바꿔도 되나요" | **고정 안전 안내문 우선** (일반 RAG보다 먼저) |
| **⑤ 무관** | 당뇨와 무관한 질문 | 범위 밖임을 안내 |

- 핵심: **①②③은 정보를 제공**하되 필요한 한계·권고를 덧붙인다(과잉 차단 금지). **④만** 일반 흐름을 멈추고 안전 안내를 앞세운다.

## 3. 안전 처리 원칙

- 진단·처방·복약(용량) 변경을 **직접 지시하지 않는다.**
- 근거가 부족하면 **추측하지 않고 보류**한다(단 RAG 3단계로 남발 방지).
- 고위험이 감지되면 일반 RAG보다 **고정 안내문을 우선**한다.
- 안전 응답도 **차갑지 않게** — "도와드릴 수 없다"가 아니라 "일반적으로 …이며, 안전을 위해 …와 상의하세요" 형태로.

## 4. 안전 안내문 (초안 — 의료전문가 검토 필요)

> 아래는 **자리표시 초안**이다. 실제 문구는 의료전문가 검토 후 `prompts/`에서 버전 관리한다.

- **복약 변경 제한 안내문 (초안):**
  "약의 용량이나 복용 방법은 사람마다 다르고 위험할 수 있어, 여기서 바꾸도록 안내드리기는 어렵습니다. 담당 의료진이나 약사와 꼭 상의해 주세요."
- **진단 제한 안내문 (초안):**
  "증상만으로 병명을 판단하기는 어렵습니다. 정확한 확인을 위해 의료진 진료를 받아보시길 권해요."
- **근거 부족 안내문 (초안):**
  "지금 가지고 있는 자료에서는 이 질문에 대한 충분한 근거를 찾지 못했어요. 정확한 정보를 위해 의료진과 상의해 주세요."
- **전문가 상담 권고문 (초안):**
  "이 부분은 개인 상태에 따라 달라질 수 있어, 담당 의료진·약사와 상의하시는 것이 가장 안전합니다."
- **응급 상황 안내문 (초안):**
  "지금 증상이 위급할 수 있어요. 즉시 담당 의료기관에 연락하거나, 응급 상황이면 119에 도움을 요청해 주세요." _(응급 문구·기준은 의료전문가 검토 필수)_

## 5. 안전성 로그

- 안전 제한 응답이 작동하면 **`safety_message_shown` 이벤트 + 질문 범주**를 기록한다(EVENT_DICTIONARY).
- 사후 검토로 고위험 질문의 빈도·유형을 파악하고, **오작동(정상 질문을 차단)·과다 트리거**를 발견해 범주·문구를 개선한다.

## 6. 넛지와 안전의 경계

- **복약 넛지는 "챙기셨어요?" 수준의 상기까지만.** 용량·종류 판단으로 확장하지 않는다.
- 넛지 문구는 공포·죄책감·강압을 쓰지 않는다(NUDGE_RULES).

## 7. 미결정 사항 (연구팀·의료전문가 확정 — Day 5 정리)

- 고위험 키워드·판별 기준의 구체 목록
- 각 안전 안내문의 최종 문구 (의료전문가 검토)
- 응급 안내 기준·연락 안내 방식
